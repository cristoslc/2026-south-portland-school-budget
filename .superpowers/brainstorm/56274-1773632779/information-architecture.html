Information Architecture

How should the site organize content for South Portland residents?

A

Question Hub

Landing page is a curated set of top questions residents are asking. Each question leads to an answer page that draws from interpretations, evidence, and persona perspectives.

Site map:
/                    → Question hub (top 8-12 questions)
/q/school-closures    → Answer page (synthesis + evidence links)
/q/teacher-cuts       → Answer page
/perspectives         → Persona picker ("I'm a...")
/perspectives/parent  → Persona briefing + open threads
/meetings             → Timeline (reference)
/meetings/2026-02-09  → Meeting detail + interpretations
/sources              → Evidence pool browser
/about                → Research disclaimer + methodology

Pros

  • Answers what people actually search for
  • SEO-friendly (question → answer)
  • Easy to share specific answers

Cons

  • Requires curating the question set
  • New questions need editorial decisions
B

Dashboard + Drill-down

Landing page is a status dashboard: key budget numbers, next meeting date, recent changes. Users drill into topics, personas, or meetings from there.

Site map:
/                    → Dashboard (key numbers + "what's new")
/topics/reconfiguration → Topic deep-dive (cross-meeting synthesis)
/topics/staffing      → Topic deep-dive
/for/parents          → Persona briefing + open threads
/for/taxpayers        → Persona briefing
/meetings             → Timeline
/meetings/2026-02-09  → Meeting detail + interpretations
/sources              → Evidence browser
/about                → Disclaimer + methodology

Pros

  • Immediate situational awareness
  • Shows "what changed" on return visits
  • Topics emerge from data, not curation

Cons

  • Dashboard needs design work to not overwhelm
  • Less shareable than specific Q&A URLs
